# 구현 계획: SPEC-TANK-001 공유 어항

> 이 문서는 구현 방향(WHAT/WHY의 실행 계획)만 다룬다. 구체적 함수/클래스/스키마 설계는 Run 단계에서 확정한다. 시간 추정치는 사용하지 않으며 우선순위 기반으로 순서를 정한다.

## Technical Approach (기술 접근)

- **프론트엔드**: React. Teams 탭 컨텍스트에서 로드. 드로잉 캔버스(HTML Canvas 기반 자유 드로잉), 어항 렌더링/애니메이션 레이어, 실시간 이벤트 수신 처리.
- **백엔드**: Node.js. Teams SSO 토큰 검증, 물고기 CRUD(생성/조회/삭제 — 수정 없음), 실시간 브로드캐스트 게이트웨이.
- **인증**: Microsoft Teams JS SDK SSO. 클라이언트에서 토큰 획득 → 서버에서 검증 → 인증된 사용자 ID를 소유자 정보로 사용.
- **실시간**: WebSocket(또는 동등 채널). 물고기 추가/삭제 이벤트를 접속 클라이언트에 push. 진입 시 스냅샷 로드 후 델타 수신.
- **영구 저장**: 데이터베이스에 물고기 레코드 영구 저장(그림 데이터, 소유자 ID, 표시 모드, 표시 이름, 생성 시각). DB 엔진은 Run 단계 결정.
- **보안 원칙**: 모든 쓰기 작업은 서버 측 토큰 검증 + 소유권 확인. 클라이언트 신뢰 금지. 그림 데이터 크기 제한/검증.

## Milestones (우선순위 기반)

### M1 — 인증 및 앱 셸 (Priority: High)

- Teams 탭 앱 부트스트랩(React) + Teams JS SDK SSO 세션 획득
- 서버 측 SSO 토큰 검증 및 인증된 사용자 신원 확보
- 커버 요구사항: REQ-AUTH-001, REQ-AUTH-002, REQ-AUTH-004, NFR-SEC-001

### M2 — 물고기 생성(드로잉) + 영구 저장 (Priority: High)

- 자유 드로잉 캔버스(undo/clear 포함), 그림 캡처
- 이름/익명 선택 등록, 유효성 검증(빈/무효 그림 거부, 크기 제한)
- DB 영구 저장(소유자 ID 내부 저장 포함)
- 커버 요구사항: REQ-DRAW-001~005, REQ-AUTH-003, REQ-OWN-001, REQ-OWN-004, REQ-PERSIST-001, REQ-PERSIST-002, NFR-SEC-003

### M3 — 어항 렌더링 + 실시간 반영 (Priority: High)

- 진입 시 전체 물고기 로드 및 헤엄 애니메이션
- 물고기 추가/삭제의 실시간 전파, 재연결/재동기화
- 커버 요구사항: REQ-DRAW-003, REQ-RT-001~004, NFR-RT-001, NFR-PERF-001

### M4 — 삭제 및 소유권 규칙 (Priority: High)

- 본인 물고기 삭제(이름/익명 공통), 타인 물고기 삭제 거부
- 삭제의 실시간 전파(REQ-RT-002와 연계)
- 커버 요구사항: REQ-OWN-002, REQ-OWN-003, REQ-OWN-005, NFR-SEC-002

### M5 — 상호작용 (Priority: Medium)

- 먹이주기(먹이 이벤트 + 반응 애니메이션)
- 물고기 클릭/호버 정보 표시(익명은 "익명"으로 표시)
- (선택) 먹이주기 효과 실시간 공유
- 커버 요구사항: REQ-INT-001, REQ-INT-002, REQ-INT-003

### M6 — 접근성 및 마무리 (Priority: Medium)

- 캔버스/상호작용 접근성 보조(키보드 경로, 라벨, 색 대비)
- 커버 요구사항: NFR-A11Y-001, NFR-CONC-001(부하/동시성 검증)

## Risks (위험 요소)

- **다수 물고기 렌더링 성능**: 물고기 수가 늘수록 프레임 저하 위험. 완화: 렌더링 최적화(캔버스 배칭/스프라이트), 동시 렌더 상한/뷰포트 컬링 검토.
- **실시간 일관성**: 연결 끊김/재연결 시 상태 불일치. 완화: 진입 스냅샷 + 델타, 재연결 시 재동기화(REQ-RT-003).
- **익명 소유권 검증**: 익명 표시와 내부 소유 추적의 분리 실패 시 오삭제/권한 누출. 완화: 서버 측 소유자 ID 검증 단일화, 익명 신원 절대 미노출(REQ-OWN-004).
- **Teams SSO 실패 처리**: 토큰 만료/실패 시 UX 붕괴. 완화: 명확한 오류 상태 + 재시도 + 기능 비활성화(REQ-AUTH-004).
- **그림 데이터 보안/용량**: 대용량/악성 그림 데이터. 완화: 크기 제한, 형식 검증, 스크립트 주입 방지(NFR-SEC-003).
- **DB 엔진 미확정**: 스키마/쿼리 패턴 지연. 완화: Run 단계 초기에 엔진 확정(`/moai db init`).
