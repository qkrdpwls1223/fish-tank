# 랜선어항 설치 안내 (동료용)

Teams에서 "랜선어항"을 쓰려면 아래 **3단계**를 한 번만 해주세요.
(회사 PC에 로컬 관리자 권한이 있어야 2단계가 됩니다.)

준비물: 담당자에게 받은 파일 2개
- `fishtank-teams.zip` (Teams 앱 패키지)
- `add-hosts.bat` (주소 등록 도우미)

---

## 1단계 — Teams에 앱 추가

1. Teams 실행 → 왼쪽 아래 **앱** 클릭
2. **앱 관리** → **앱 업로드** → **사용자 지정 앱 업로드**
3. 받은 `fishtank-teams.zip` 선택 → 추가

> "사용자 지정 앱 업로드" 항목이 안 보이면 회사 정책상 막힌 것입니다.
> 그 경우 담당자에게 문의하세요.

## 2단계 — 주소 등록 (add-hosts.bat 더블클릭)

1. `add-hosts.bat` 를 **더블클릭**
2. "이 앱이 기기를 변경하도록 허용하시겠어요?"(UAC) → **예**
3. "[완료] hosts 에 다음 줄을 추가했습니다" 메시지가 뜨면 성공
4. 창을 닫으세요

> 파일 실행 시 SmartScreen 경고가 뜨면 **추가 정보 → 실행**을 누르세요.
> UAC 에서 관리자 비밀번호를 요구하는데 없다면, 회사 IT 에 문의해야 합니다.

## 3단계 — 접속 확인

1. Teams에서 방금 추가한 **랜선어항** 앱 열기
2. 잠시 후 물고기 어항 화면이 나오면 완료입니다

> **"연결이 거부되었습니다" 또는 인증서 경고**가 뜨면:
> 브라우저에서 `https://fishtank.fllab.internal` 접속해 자물쇠(정상)인지 확인하세요.
> 경고가 뜨면 인증서 신뢰가 안 된 것이라 담당자에게 알려주세요.

---

## 제거하려면

`remove-hosts.bat` 를 더블클릭하면 2단계에서 추가한 주소가 제거됩니다.
Teams 앱은 앱 관리에서 삭제하면 됩니다.

## 자주 묻는 것

| 증상 | 해결 |
|------|------|
| "사용자 지정 앱 업로드" 안 보임 | 회사 정책 — 담당자 문의 |
| bat 실행 시 관리자 권한 없음 | 로컬 관리자 권한 필요 — IT 문의 |
| 앱은 열리는데 "연결 거부" | 2단계(add-hosts) 실행 여부 확인 |
| 인증서 경고 | 담당자에게 루트 CA 설치 요청 |
| 로그인 실패 | Teams 안에서 열었는지 확인(브라우저 직접 접속은 안 됨) |
